About the Web Conference 2018 app

Its data structure follows the newly published ScholarlyData Ontology, formerly Semantic Web Conference Ontology, that describes persons, organizations, publications and events of the conference. The application allows browsing these data. It is also a client-side mashup that can enrich them using other data sources (DBLP, ad hoc endpoint) or search engines (Google, DuckDuckgo).

You can use it offline to browse the conference dataset. You will need an Internet connection for viewing data from online sources, as well as to vote for your preferred poster and demo.

Periodically (but not too often), you can update the conference dataset in the Tools menu item.

You can create an account on our server to enrich your data in the app, as well as to be able to participate in the vote.

How it works

In 2017, the app architecture has been completely refactored:

  • The conference dataset now follows the ScholarlyData ontology and has been issued using cLODg (conference Linked Open Data generator) V2.
  • Its core engine is based on RDF data (instead of JSON), internally served using the rdflib.js library. Feel free to open the console and watch the app sending SPARQL requests to the store.
  • It fully runs on the client side, and relies on the Angular 2 JavaScript framework, using Web components and other advances. The design layer uses Angular Material.
